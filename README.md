# ノードイベントログ 仕様

## 概要

本アプリで扱うログは、各ノードで発生したイベント(解像度の要求・応答受信など)を時系列に記録したものです。
1行が1イベントに対応し、1行につき5つの値が区切り文字(カンマ・タブ・空白のいずれか)で区切られています。

## フォーマット

```
<時刻> <ノードID> <セグメント番号> <コード値> <種別>
```

### 列の定義

| # | 列名       | 内容                                   | 形式・例                              |
|---|------------|----------------------------------------|----------------------------------------|
| 1 | 時刻       | イベント発生時刻                       | `hh:mm:ss.SSS`(例: `15:15:02.875`)     |
| 2 | ノードID   | イベントを発生させたノードの識別子     | `0`〜`49`(通常ノード)、`-1`(上位ノード) |
| 3 | セグメント番号 | 対象セグメントの番号                | 3桁の数値文字列(例: `000`, `035`)      |
| 4 | コード値   | イベントに紐づく値(解像度を表す)       | `720` / `480` / `360` / `240` のいずれか |
| 5 | 種別       | イベントの種類                         | `creq` / `preq` / `pres`(下表参照)      |

## ノードIDの区分

| ノードID | 区分         | 備考                                            |
|----------|--------------|-------------------------------------------------|
| `0`〜`49`  | 通常ノード   | 最大50台。種別は基本的に `creq` のみ            |
| `-1`     | 上位ノード   | 1台のみ。要求(`preq`)・応答受信(`pres`)の両方を記録 |

## コード値と表示色の対応

アプリ上では、コード値に応じて以下の色で表示します。

| コード値 | 意味(解像度) | 表示色     |
|---------|--------------|------------|
| `720`   | 720p         | 青         |
| `480`   | 480p         | オレンジ   |
| `360`   | 360p         | 緑         |
| `240`   | 240p         | 赤         |

上記以外の値が入っていた場合は「その他」としてグレー表示になります。

## 種別(5列目)の定義

| 値     | 意味                             | 対象ノード       |
|--------|----------------------------------|------------------|
| `creq` | クライアントからの要求           | 通常ノード(0〜49) |
| `preq` | 上位ノードへの要求               | 上位ノード(-1)    |
| `pres` | 上位ノードからの応答受信         | 上位ノード(-1)    |

## サンプル

```
15:15:02.875 26 000 240 creq
15:15:02.876 27 000 240 creq
15:15:02.906 -1 000 480 preq
15:15:02.906 -1 000 360 preq
15:15:02.907 -1 000 720 preq
15:15:03.587 41 000 240 creq
```

## 備考

- 時刻は先頭のレコードを基準(0秒)とした相対経過秒数に変換して扱う
- 同一ノード・同一セグメントに対して、ごく短時間(数ミリ秒以内)に複数の異なるコード値のイベントが記録されることがある(例: 上位ノードでの `preq` の連続発行)
